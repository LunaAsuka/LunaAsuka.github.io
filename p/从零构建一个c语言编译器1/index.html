从零构建一个c语言编译器(1)

Building a C compiler!（整体设计）

May 11, 2022
阅读时长: 2 分钟

编译器，大多数人理解的是可以将某种语言的源代码文件转换成一个可执行文件的程序，日常来说确实如此，但是严格来讲编译器只是负责将一种语言翻译成另外一个语言，例如将 c 翻译成汇编。

而该项目所说的编译器其实是一个 c 语言解释器，我们要做的工作是先将 c 语言翻译成我们自己设计的一种伪汇编，然后在一个我们自己模拟的虚拟机环境下直接解释运行伪汇编。不过就算这样我们一样做了“编译”这个工作对吧。

这样做的理由如下：

1.我还没系统的学过汇编，让我直接把 c 翻译成汇编是不可能的事。

2.这种设计能够大大简化我们所需的工作，并且依然能够得到令人满意的结果。

#include <stdio.h>
#include <stdlib.h>
#include <memory.h>
#include <string.h>
#include <unistd.h>
#include <stdint.h>

int token;//当前的标识符
char* src;//源文件，即我们要编译的文件
int memorySize;
int line;//行号，Debug用的
int debug;//调试模式

void next()
{
    token = *src++;
    //do something
    return ;
}

void expression(int level)
{
    //do something
}

void parse()
{
    next();
    while(token > 0)
    {
        //do something
    }
}

int VM()
{
    //do something
    return 0;
}

int main(int argc,char** argv)
{
 	int tmp;//标识一些函数返回状态的临时变量
    int fd;//文件描述符
    argc--;//main函数的参数中第一个是main自己，需要去掉这个，所以数量-1
    argv++;//参数向后推一个，去掉main
    memorySize = 512 * 1024;//为各个段分配的内存大小,521KB
    line = 1;

    if (argc > 0 && (*argv)[0] == '-' && (*argv)[1] == 'd')
    {//开启Debug模式
        debug = 1;
        argc--;
        argv++;
    }

    if (argc < 1)
    {//参数过少
        printf("usage: ./718Compiler [-d] xxx.c\n");
        return -1;
    }

    if((fd = open(*argv,0)) < 0)
    {//打开源文件
        printf("could not open(%s)\n",*argv);
        return -1;
    }

    if(!(src = (char*)malloc(memorySize)))
    {//给src分配空间
        printf("could not malloc(%d) for source area\n",memorySize);
    }

    if((tmp = read(fd,src,memorySize - 1)) <= 0)
    {//把源文件读入到src中
        printf("read source file failed\n");
        return -1;
    }
    src[tmp] = 0;//向src末尾添加'\0'作为EOF标志
    close(fd);

    parse();
    return VM();
}

我会尽可能详细的做全注释，以确保初学者也能轻松的看懂。

我们可以看到代码中有几个尚未实现的函数，这些函数我们会在后续的文章中逐步扩充，现在不需要去关注它们，专注于目前的代码就好。

OK，以上就是本节的内容，实际上也没什么东西，大体表示出了整个编译器的框架，接下来我们会逐步填充这个不完整的程序，最终构建起我们的编译器。

ps：注意我们的代码都是运行在32位环境下，64位编译环境下可能会发生一些未知的错误，所以我推荐你在用 gcc 编译的时候加上 ‘-m32’ 选项。
